# AI 개발 사이드프로젝트 가이드 (Docker + AI 서비스)

> **목표**: Docker 기반의 일관된 개발/배포 환경 위에 간단한 AI 웹 서비스를 구현하고, 대학 과제 형태로 **체계적인 산출물**(보고서, 로그, 코드, 실험기록, 릴리즈)을 남긴다. 이 문서는 AI에게 그대로 입력해 **작업 규칙과 단계**를 강제하기 위한 **SOP(Standard Operating Procedure)** 이다.

---

## 0) 전체 산출물 목록 (제출 패키지)
- `README.md` — 프로젝트 개요/실행 방법/아키텍처 다이어그램
- `REPORT.md` — 과제 보고서(문제정의, 방법, 결과, 토론, 한계/윤리)
- `CHANGELOG.md` — 버전별 변경내역 (Keep a Changelog 스타일)
- `EXPERIMENTS.md` — 실험 설계·결과·해석 표준 양식
- `RISKS.md` — 리스크/이슈 로그 및 대응
- `docker/` — Dockerfile, docker-compose.yml, Entrypoint/헬스체크 스크립트
- `src/` — 서비스 소스코드 (예: FastAPI/Flask)
- `models/` — 모델/토크나이저(가능하면 가벼운 샘플, 대형모델은 다운로드 스크립트)
- `tests/` — 단위/통합 테스트
- `Makefile` — 자주 쓰는 명령어 집합(빌드/테스트/실행/린트)
- `.env.example` — 필요 환경변수 예시(비밀값 제외)
- `requirements.txt` 또는 `pyproject.toml`
- `LICENSE` — 오픈소스 라이선스 표기(기본 MIT)
- `docs/` — API 문서(예: OpenAPI), 아키텍처/ERD/시퀀스 다이어그램, 스크린샷

## 1) 프로젝트 규칙 (AI가 반드시 준수)
1. **작업저널 규칙**
   - 모든 작업은 **시간 스탬프**와 함께 `logs/daily/<YYYY-MM-DD>.md`에 기록.
   - 각 기록은 *목표 → 수행내용 → 결과물 링크 → 문제/해결 → 다음 액션* 순서로 작성.
2. **브랜치/커밋 규칙**
   - 메인: `main`, 기능: `feature/<주제>`, 실험: `exp/<주제>`
   - 커밋 메시지: `type(scope): summary` (예: `feat(api): add /predict endpoint`)
   - `type` 목록: feat, fix, docs, refactor, test, chore, build, ci.
3. **이슈/태스크 규칙**
   - 각 태스크는 **명확한 DoD(Definition of Done)** 포함.
   - 이슈 템플릿: *배경/목표/수행항목/DoD/리스크/예상시간*.
4. **보안/비밀정보**
   - 실 비밀키는 저장 금지. `.env`는 로컬에만, 리포엔 `.env.example`만.
   - 로그/스크린샷 업로드 전 식별정보 마스킹.
5. **재현성**
   - `Makefile`과 Docker만 있으면 **로컬/CI에서 동일 실행** 가능해야 함.
   - 결과 재현을 위한 **실험 파라미터/버전/랜덤시드** 기록 의무.
6. **문서화**
   - 변경 시 `README.md`/`docs/`/`CHANGELOG.md` 동시 갱신.
   - 모든 공개 API는 예시 요청/응답 포함.
7. **품질 게이트**
   - `make lint test` 통과 없이는 `main` 병합 금지.
   - 컨테이너 **헬스체크** 반드시 구현.

## 2) 단계별 계획 (AI 실행 체크리스트 & 프롬프트)

### Phase A. 킥오프 & 문제정의 (0.5일)
**목표**: 과제 목표/범위/성공지표 확정, 폴더/라이선스/기본 문서 생성.

**체크리스트**
- [ ] 문제정의: 대상 문제, 사용자 페르소나, 기대효과, 제약(시간/자원)
- [ ] 기능범위(MVP): 입력/출력/품질지표(지연시간, 정확도 등)
- [ ] 시스템 개요 다이어그램 초안 작성
- [ ] 리포 초기화: 구조/라이선스/README 스캐폴딩

**프롬프트 예시**
```text
너는 대학 과제 조교 역할의 AI다. 다음 정보를 바탕으로 프로젝트 킥오프 산출물을 작성해라.
...
````

### Phase B. 환경 구축 (1일)

**목표**: Docker 이미지/컴포즈/Makefile로 재현 가능한 개발환경 완성.

**체크리스트**

* [ ] Dockerfile 작성(멀티스테이지)
* [ ] docker-compose.yml 구성
* [ ] Makefile 작성
* [ ] 헬스체크/엔트리포인트 구현
* [ ] `.env.example` 제공

### Phase C. AI 서비스 구현 (1~2일)

**목표**: FastAPI 기반 `/health`, `/predict` 구현.

### Phase D. 실험 및 성능검증 (0.5~1일)

**목표**: 벤치마크 실행, 결과 기록, 개선안 도출.

### Phase E. 리포팅 & 릴리즈 (0.5일)

**목표**: 보고서 작성, 릴리즈 태깅, 제출물 패키징.
